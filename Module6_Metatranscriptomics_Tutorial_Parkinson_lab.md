    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-metapro-metatranscriptomics-practical-lab" class="anchor" aria-hidden="true" href="#metapro-metatranscriptomics-practical-lab"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>MetaPro Metatranscriptomics Practical Lab</h1>

<p><strong>This tutorial was produced by Billy Taj (<a href="mailto:billy.taj@sickkids.ca">billy.taj@sickkids.ca</a>), Mobolaji Adeolu (<a href="mailto:adeolum@mcmaster.ca">adeolum@mcmaster.ca</a>), John Parkinson (<a href="mailto:john.parkinson@utoronto.ca">john.parkinson@utoronto.ca</a>) &amp; Xuejian Xiong (<a href="mailto:xuejian@sickkids.ca">xuejian@sickkids.ca</a>)</strong></p>
<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview</h2>
<p>This tutorial will take you through the MetaPro pipeline for processing metatranscriptomic data. The pipeline, developed by the Parkinson lab, consists of various steps which are as follows:</p>
<ol>
<li>Remove low-quality reads that MetaPro has low confidence in processing.</li>
<li>Remove adapter sequences, which are added during library preparation and sequencing steps, and trim low quality bases and sequencing reads.</li>
<li>Remove duplicate reads to reduce processing time for following steps.</li>
<li>Remove vector contamination (reads derived from cloning vectors, spike-ins, and primers).</li>
<li>Remove host reads (if exploring a microbiome in which the host is an issue).</li>
<li>Remove abundant rRNA sequences which typically dominate metatranscriptomic datasets despite the use of rRNA removal kits.</li>
<li>Add duplicated reads, removed in step 2, back to the data set to improve quality of assemblies.</li>
<li>Classify reads to known taxonomic groups and visualize the taxonomic composition of your dataset.</li>
<li>Assemble the reads into contigs to improve annotation quality.</li>
<li>Annotate reads to known genes.</li>
<li>Map identified genes to the swiss-prot database to identify enzyme function</li>
<li>Generate normalized expression values associated with each gene.</li>
<li>Visualize the results using KEGG metabolic pathways as scaffolds in Cytoscape.</li>
</ol>
<p>The MetaPro metatranscriptomic pipeline includes existing bioinformatic tools and a series of Python scripts that handle the orchestrated invocation of the bioinformatics tools, read-conflict resolution, file format conversion, and output parsing. We will go through these steps to illustrate the complexity of the process and the underlying tools and scripts.  The operation of the pipeline is designed to run all steps automatically in succession.  However, for the purposes of the tutorial, a single-step tutorial-mode has been included in MetaPro's design.  One can now call the MetaPro pipeline to perform each step in isolation, for learning purposes.</p>
<p>New, faster, and/or more accurate tools are being developed all the time, and it is worth bearing in mind that any pipelines need to be flexible to incorporate these tools as they get adopted as standards by the community. For example, over the past two years, our lab has transitioned from Trimmomatic to AdapterRemoval, and from BLAST to DIAMOND.
Note:  This workshop was designed for use with DIAMOND v0.826.  Newer versions of DIAMOND will be incompatible with the pre-compiled database files we have made as part of this exercise.<br>
To illustrate the process we are going to use sequence reads generated from the contents of the colon of a mouse. These are 150 bp single-end reads. Paired-end reads can also be used, and are often preferred because they can improve annotation quality when there is enough overlap in the read pairs to improve the effective average read length. Working with paired-end data involves an additional data processing step (merging of overlapping reads) produces more files during data processing (files for merged/singleton reads, forward reads, and reverse reads), but the structure of a pipeline for paired-end data is similar to the pipeline described here and can be readily adapted.</p>
<p>Rather than use the entire set of 25 million read, which might take several days to process on a desktop, the tutorial will take you through processing a subset of 100,000 single-ended reads.</p>
<p><strong>Note</strong>
The purpose of this tutorial is to demonstrate MetaPro's various steps.  The pipeline is fully capable of running all of the steps without the intervention of the user beyond an initial call to the program.  If you wish to simply use MetaPro, do not use the --tutorial option.
This tutorial also assumes that the pipeline files are contained in a directory</p>

Remove abundant rRNA sequences</h3>
<p>rRNA genes tend to be highly expressed in all samples and must therefore be screened out to avoid lengthy downstream processing times for the assembly and annotation steps. MetaPro uses [Barrnap] (<a href="https://github.com/tseemann/barrnap">https://github.com/tseemann/barrnap</a>) and [Infernal] (<a href="http://infernal.janelia.org/" rel="nofollow">http://infernal.janelia.org/</a>).
You could use sequence similarity tools such as BWA or BLAST for this step, but we find Infernal, albeit slower, is more sensitive as it relies on a database of covariance models (CMs) describing rRNA sequence profiles based on the Rfam database. Due to the reliance on CMs, Infernal, can take as much as 4 hours for ~100,000 reads on a single core.  In an effort to shrink the computing time, we leverage a computing cluster's multiple cores.
Here, MetaPro demonstrates the case for automation.
MetaPro subdivides the input data, coordinates the concurrent processes, and collects the results into one single file after all of the scanning has been complete.</p>
<p>MetaPro will perform the following:</p>
<ul>
<li>subdivide the input data in user-defined chunksizes.</li>
<li>Each chunk is then run independently:</li>
<li>
<ul>
<li>run each chunk through Barrnap</li>
</ul>
</li>
<li>
<ul>
<li>using the results of Barrnap, filter the data chunk into mRNA, and leftover data for further scanning.</li>
</ul>
</li>
<li>
<ul>
<li>run each leftover chunk through Infernal.</li>
</ul>
</li>
<li>
<ul>
<li>filter the Barrnap leftover chunk using the Infernal results, to get mRNA, and "other"</li>
</ul>
</li>
<li>collect all of the data pieces (Barrnap mRNA, Infernal mRNA) into mRNA, and "other"</li>
</ul>
<p>By running things this way, the rRNA step takes 4 minutes(as recorded with a 40-core computing node with 200 GB RAM, and an rRNA chunksize of 1000 reads), but it requires significant computing power, memory, and storage space, not available on a typical desktop PC.
If you were to run this on your own, you will need the RFam database.

Annotate reads to known genes/proteins</h3>
<p>Here we will attempt to infer the specific genes our putative mRNA reads originated from. In our pipeline we rely on a tiered set of sequence similarity searches of decreasing accuracy - BWA, BLAT, and DIAMOND. While BWA provides high stringency, sequence diversity that occurs at the nucleotide level results in few matches observed for these processes. Nonetheless it is quick. To avoid the problems of diversity that occur at the level of nucleotide, particularly in the absence of reference microbial genomes, we use a cascaded method involving two other tools: BLAT, and DIAMOND. BLAT provides a more sensitive alignment, along with quality scores to rank the matches.  DIAMOND is used to provide more sensitive peptide-based searches, which are less prone to sequence changes between strains.</p>
<p>Since BWA and BLAT utilize nucleotide searches, we rely on the [ChocoPhlan pangenome database] (<a href="http://huttenhower.sph.harvard.edu/humann2_data/chocophlan/" rel="nofollow">http://huttenhower.sph.harvard.edu/humann2_data/chocophlan/</a>) that we obtained from The Huttenhower lab, which contains over 10000 organisms in separate .ffn files.  We create a merged copy of these sequences, and index it for BWA to use.  We leave it in its separated state for BLAT to use.</p>
<p>For DIAMOND searches we use the [Non-Redundant (NR) protein database] (ftp://ftp.ncbi.nih.gov/blast/db/FASTA/nr.gz) from the NCBI.</p>
<p>This is a computationally intensive step.  We employ our subdivision strategy here, similar to our design for rRNA removal, as seen below</p>
<ul>
<li>The mRNA read data (contigs, remaining singletons, and remaining paired reads if applicable) are split into chunksizes (GA_chunksize in the configuration)</li>
<li>Each chunk is sent through BWA to be aligned against the ChocoPhlan database</li>
<li>A map of genes to constituent reads is formed for each chunk.</li>
<li>Reads not annotated by BWA are isolated, and are sent through to BLAT.</li>
<li>Another map of genes to constituent BLAT-annotated reads are formed for each chunk.</li>
<li>Reads not annotated by BLAT are isolated, and are sent through to DIAMOND.</li>
<li>A 3rd batch of maps of annotations to constituent reads are formed for the DIAMOND reads for each chunk.</li>
</ul>
<p>In all, this leaves us with (split into chunks):</p>
<ul>
<li>a batch of BWA-annotated gene-to-read maps</li>
<li>a batch of BLAT-annotated gene-to-read maps</li>
<li>a batch of DIAMOND-annotated protein-to-read maps</li>
<li>a batch of reads unannotated by BWA, BLAT, and DIAMOND.</li>
</ul>
<p>These batches of files are then sent to a custom script that will perform all of the final merging:</p>
<ul>
<li>collect and merge every map to form one map.</li>
<li>collect and merge the unannotated DIAMOND reads</li>
<li>collect all of the genes that were found in the reads, and convert them into proteins.  Then merge them with the proteins found in DIAMOND.  This step is for downstream analysis.</li>
</ul>

MetaPro combines the predictions of all 3 tools to give 2 answers, a lower-confidence set of enzyme predictions, and a higher-confidence set of predictions.

<p><strong>Notes</strong>:</p>
<ul>
<li>
<p>MetaPro's high-confidence and low-confidence are determined by the following:</p>
<ul>
<li>DIAMOND: Low-confidence hits are ones with an e-value of 1e-5 or smaller.  High-confidence hits are ones with an e-value of 1e-10 or smaller</li>
<li>PRIAM: Low-confidence hits are ones with e-values lower than 1e-5.  High-confidence hits are ones where their probability value is 0.5 or higher</li>
<li>DETECT: There is no separation.</li>
</ul>
</li>
<li>
<p>MetaPro reconciles the 3 annotations in the following manner:</p>
<ul>
<li>The Enzymes predicted by DETECT are taken, followed by the annotations that agree between PRIAM and DIAMOND</li>
<li>In the event of multiple enzymes being annotated to the same protein:
<ul>
<li>Every enzyme annotation comes with a probability score.</li>
<li>MetaPro includes a enzyme co-occurence database (compiled from the ENZYME database) that contains pairs of enzymes known to exist together</li>
<li>Using this co-occurence database, MetaPro filters invalid predictions.
<ul>
<li>In cases where more-than-2 enzymes are annotated to a protein, the top 2 enzymes are taken, based on the probability score.</li>
<li>If a pair of enzymes do not exist in the database, the enzyme with the higher probability score is declared the proper annotation.</li>
<li>Otherwise, the annotation is declared as a pair of enzymes.</li>
</ul>
</li>
</ul>
</li>
</ul>
</li>
</ul>
